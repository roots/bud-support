## bud build --cache

The `--cache` flag is recommended for outstanding performance. Compare the `compiled in` output to the uncached build (above).

<CodeBlock title='$ yarn bud build --cache' highlight={6}>{BuildCache}</CodeBlock>

## bud build --target

In multi-compiler configurations, many compilers are run in a batch.

<CodeBlock title='$ yarn bud build'>{BuildMulti}</CodeBlock>

The `--target` flag will only compile a specified compiler.

<CodeBlock title='$ yarn bud build --target theme'>{BuildTargetTheme}</CodeBlock>

It can be passed more than once.

<CodeBlock title='$ yarn bud build --target theme --target plugin'>{BuildTargetThemePlugin}</CodeBlock>

## bud build --hash

The `--hash` flag will enable filename hashing for the build.

<CodeBlock title='$ yarn bud build --hash'>{BuildHash}</CodeBlock>

## bud build --ci

The `--ci` flag uses webpack's default cli output (for interoperability).

<CodeBlock title='$ yarn bud build --ci'>{BuildCI}</CodeBlock>
